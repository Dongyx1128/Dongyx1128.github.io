Ubuntu 下文件的安装与卸载命令汇总

dpkg 即 Debian package 的简写 ，是 Debian 和基于 Debian 的系统中一个主要的包管理工具，可以用来安装、构建、卸载、管理 .deb 格式的软件包。dpkg 的主要和更友好的前端是 aptitude 。dpkg 本身完全由命令行参数控制，命令行参数只包含一个动作和零个或多个选项。动作参数告诉 dpkg 要做什么，选项以某种方式控制动作的行为。
　　
　　dpkg 主要是用来安装已经下载到本地的 deb 软件包，或者对已经安装好的软件进行管理。

apt 和 apt-get 是类似红帽系的 yum 工具，能够自行解决 deb 包的依赖关系。直接从远程的软件仓库下载安装软件。

2. dpkg 命令安装与卸载

dpkg -i|--install：手动安装软件包(这个命令并不能解决软件包之前的依赖性问题)，如果在安装某一个软件包的时候遇到了软件依赖的问题,可以用 apt-get -f install 解决信赖性这个问题。

dpkg -I|--info ：查询deb包的详细信息，在一个软件包下载到本地之后看看用不用安装。

dpkg -l|--list：列出当前系统中所有的包，可以和参数less一起使用在分屏查看。 (类似于rpm -qa)

dpkg -L|--listfiles：查询系统中已安装的软件包所安装的位置。 (类似于rpm -ql)

dpkg -l |grep -i "软件包名" ：查看系统中与”软件包名”相关联的包。

dpkg -s|--status：查询已安装的包的详细信息。

dpkg -S|--search：查询系统中某个文件属于哪个软件包。(类似于rpm -qf)

dpkg -r|--remove：卸载软件包，不是完全的卸载，它的配置文件还存在。

dpkg -P|--purge：全部卸载，但是还是不能解决软件包的依赖性的问题。

dpkg -c|--contents：显示包的目录内容

dpkg -reconfigure：重新配置。

Ubuntu 下文件的打包、解压缩命令汇总

在 Ubuntu 操作系统中，文件压缩与解压是日常操作中常见的任务之一。本文将介绍七种常用的压缩和解压工具：tar、gzip、bzip2、rar 、 zip、xz 和 Z，在 Ubuntu 中的使用方法和常见场景。

tar只是打包工具！不是解压缩工具！bzip2 比 gzip 压缩率更高，gzip 比 bzip2 压缩更快。压缩工具分别为 bzip2( j ) 和 gzip( z ) 。通常 .zip 占用空间比 .tar.gz 大。

1. tar

tar 是一个常用的归档工具，在 Linux 和类 Unix 系统中广泛使用。它的名称来自于 “tape archive”（磁带归档）的缩写，最初设计用于在磁带上创建归档备份。现在，tar 主要用于创建和管理归档文件，通常用于将多个文件或目录打包成单个文件，以便于存储、传输或备份。

下面是关于 tar 的一些重要信息和常用命令：

1. 基本语法

tar 命令的基本语法如下：

  • options：是一些选项，用于指定 tar 命令的行为，比如 -c（创建归档）、-x（解压归档）、-v（显示详细信息）等。
  • archive_name：是要创建或操作的归档文件的名称。
  • files_or_directories：是要打包或解包的文件或目录的名称。

2. 常用选项

  • -c, --create：创建归档文件。打包c。-cvf。
  • -x, --extract：解压归档文件。解包x。-xvf。
  • -v, --verbose：显示详细信息，包括处理的文件列表。
  • -f, --file：指定归档文件的名称。
  • -z, --gzip：使用 gzip 压缩/解压缩归档文件。压缩 -czvf。解压 -xzvf。
  • -j, --bzip2：使用 bzip2 压缩/解压缩归档文件。压缩 -cjvf。解压 -xjvf。
  • -t, --list：列出归档文件中的内容。
  • -r, --append：向归档文件中追加文件。
  • -u, --update：仅向归档文件中添加新的或更新的文件。
  • -C, --Change：指定解压缩的目标目录。
  • -A, --catenate, --concatenate：将多个归档文件合并成一个。

2. gzip

简介： gzip 是一个常用的文件压缩工具，文件经它压缩过后，其名称后面会多出 .gz 的扩展名。

1. 基本语法

gzip 命令的基本语法如下：

  • [-acdfhlLnNqrtvV]：是一些选项，用于指定 gzip 命令的行为。
  • filename：是要创建或操作的文件名称。

2. 常用选项

  • -a, --ascii：使用ASCII文字模式。
  • -c, --stdout：把压缩后的文件输出到标准输出设备，不去更动原始文件。
  • -d, --decompress：强制执行解压。
  • -f, --force ：强行压缩文件。不理会文件名称或硬连接是否存在以及该文件是否为符号连接。
  • -l, --list：列出压缩文件的相关信息。
  • -n, --no-name：压缩文件时，不保存原来的文件名称及时间戳记。
  • -N, --name：压缩文件时，保存原来的文件名称及时间戳记。
  • -q, --quiet：不显示警告信息。
  • -r, --recursive ：递归处理，将指定目录下的所有文件及子目录一并处理。
  • -t, --test：测试 .gz 压缩文件的完整性。
  • -v, --verbose：显示详细信息，包括处理的文件列表。
  • -V, --version：显示版本信息。
  • -L, --license：显示版权信息。
  • -h, --help：显示帮助。

3. bzip2

简介： bzip2 是另一个常用的文件压缩工具，采用新的压缩演算法，它可以生成更小的压缩文件。若没有加上任何参数，bzip2压缩完文件后会产生 .bz2 的压缩文件，并删除原始的文件。

1. 基本语法

bzip2 命令的基本语法如下：

  • [-cdfhkLstvVz]：是一些选项，用于指定 bzip2 命令的行为。
  • filename：是要创建或操作的文件名称。

2. 常用选项

  • -c, --stdout：将压缩与解压缩的结果送到标准输出。
  • -d, --decompress：强制执行解压。
  • -z, --compress：强制执行压缩。
  • -f, --force ：bzip2在压缩或解压缩时，若输出文件与现有文件同名，预设不会覆盖现有文件。若要覆盖，请使用此参数。
  • -k, --keep：bzip2在压缩或解压缩后，会删除原始的文件。若要保留原始文件，请使用此参数。
  • -s, --small ：降低程序执行时内存的使用量。
  • -t, --test：测试 .bz2 压缩文件的完整性。
  • -v, --verbose：显示详细信息，包括处理的文件列表。
  • -V, --version：显示版本信息。
  • -L, --license：显示版权信息。
  • -h, --help：显示帮助。
